# Domain Docs

Workgrove is a single-context repository.

Before exploring or designing, read:

- `CONTEXT.md`
- Relevant architectural decisions under `docs/adr/`, when present

Use the glossary’s canonical vocabulary in issue titles, specifications, tests, and implementation discussions. If a proposed change contradicts an existing architectural decision, surface the conflict explicitly.

Domain documentation is created lazily:

- `CONTEXT.md` contains domain language only.
- `docs/adr/` contains hard-to-reverse architectural decisions resulting from real trade-offs.
